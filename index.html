個人專案

報名小狸🏸 - 羽球LINE Bot

　　兩年前，我開始接觸羽球運動，發現每次報名時，大家都是透過 LINE 群組接龍的方式進行。然而，當報名人數眾多時，經常會出現遺漏或重複登記的情況，給球友和團主都帶來了不少麻煩。當時我正好也在研究 LINE Bot 的開發技術，於是決定結合自己的興趣和專業知識，開發出一個羽球報名機器人，希望能夠解決報名過程中的種種困難，提供一個更高效的解決方案。

需求背景

球友端痛點：
  • 容易遺漏訊息：群組訊息量大，報名資訊容易被其他聊天內容淹沒，導致球友錯過報名時間或資訊。
  • 場次資訊難以查詢：當有多場次選擇時，球友很難快速找到適合自己的場次資訊，如場地、時間等，經常需要翻找群組歷史訊息。
  • 臨時取消聯繫困難：當球友臨時有事需要取消報名時，經常找不到團主或無法及時通知，導致取消資訊未能及時更新，造成不必要的困擾。
團主端痛點：
  • 手動整理報名資訊耗時：每次需要手動整理報名的成員名單、確認人數，管理工作量大且容易出現錯誤。
  • 報名與取消統計困難：當球友需要更改報名狀態或取消報名時，團主需手動更新名單，無法即時獲取最新的報名情況。
  • 多場次管理困難：對於同時開設多場次的團主，手動管理每個場次的報名名單與細節會變得更加複雜且耗時，容易出現資訊混淆和疏漏。
專案發想：
  • 選擇 LINE Bot 的原因：
    • LINE 在全台灣擁有超過 2000 萬用戶，覆蓋率高。
    • LINE 具有基本的身份認證功能，註冊流程簡單方便。
    • 團主可輕鬆開設羽球場次，球友也能快速報名，減少繁瑣手續。
  • 技術應用：
    • Google Firebase：用於儲存報名及場次資料，確保數據的安全性與即時性。
    • LINE LIFF：實現更多自由化且彈性的前端介面，提供直觀、方便的操作體驗。
  • 整體目標：
    • 提升報名流程的便捷性與使用者體驗。
    • 確保數據管理的高效性與可靠性。

專案重點

使用技術：
  • 後端：Python FastAPI、Redis、Google Firebase
  • 前端：HTML、JavaScript、CSS
主要功能：
  • 球友端：
    1. 查看場次：瀏覽所有已開放的場次，了解場地、時間、人數等資訊。
    2. 已報場次管理：查看自己已報名的場次，可查看目前名單亦可隨時取消報名，避免占用名額。
    3. 快速聯繫團主：點擊『團主資訊』可直接跳轉到團主LINE。
    4. 場次提醒：利用LINE Notify，如正取名單有球友取消，候補上了會有提醒通知。
  • 團主端：
    1. 快速開場次：設定場次名稱、時間、地點及人數限制，一鍵生成場次資訊。另外有提供『週期場』的設定，會自動生成每一週的場次。
    2. 編輯場次設定：隨時調整場次的各項細部設定，確保場次資訊保持最新。
    3. 球友名單管理：實時查看並管理各個場次的報名名單，方便確認出席者。
    4. 黑名單管理：團主可以將不符合規範的球友加入黑名單，限制其報名參加未來場次的權限。
預期效益：
  • 簡化報名流程：讓團主與球友都能告別以往繁瑣的群組接龍報名方式，透過自動化的報名系統，減少手動管理的時間和精力。
  • 資訊透明化：提供即時更新的報名資訊，確保球友與團主之間的資訊對稱，避免過去因訊息不一致而造成的誤解或錯過報名。
  • 提升效率：預計能減少團主手動整理報名資料的時間約 50%，同時讓球友能更快速地完成報名，減少重複確認的溝通成本。
  • 增進使用者體驗：通過更直觀、易操作的報名介面，提升球友的參與感與滿意度，使得報名流程更加順暢。
未來規劃：

LineTion

　　為了解決在職場中常見的問題諸如「不知道要找誰詢問」、「問錯人反而讓問題更複雜」等，我們發現這些情況會影響工作效率。因此，我們開發了 LineTion 平台，一個結合 LINE Bot 與 Notion 資料庫的互動平台，旨在提供員工一個能快速查詢和提交問題的解決方案。

需求背景

工作環境痛點：
  • 員工常因不確定要詢問誰而浪費大量時間。
  • 問錯人導致解決問題的時間拉長。
  • 需要有效率地管理和分享內部知識，避免重複詢問。
專案發想：
  • 將 LINE Bot 的即時互動性與 Notion 的資料管理能力結合，打造一個「問題」與「解答」的互動平台。
  • 利用 Notion 作為後端資料庫，讓知識庫可視化且易於維護。

專案重點

使用技術：

Python FastAPI、Notion API、LINE Messaging API。

主要功能：
  1. Q&A 查詢功能：
    • 使用者可透過 LINE Bot 查詢常見問題，Bot 會自動回覆相關解答。
    • 使用 Notion 資料庫管理 Q&A 內容，實現靈活的資料更新。
  2. 案件提交與處理：
    • 使用者可提交異常問題，系統自動分類並匯入 Notion。
    • 案件狀態更新自動通知使用者，包括「處理中」和「已完成」狀態。
  3. 案件進度查詢：
    • 支援根據案件編號或查詢全部案件，實時獲取處理狀態。
  4. 數據視覺化：
    • 每日自動撈取 Notion 資料庫的數據，製作案件趨勢圖及分類統計圖，顯示在後台首頁。
預期效益：
  • 加速案件處理流程，預計能減少 40% 的處理時間。
  • 建立公司內部的知識庫，讓用戶能夠自行解決常見問題。
  • 利用數據分析提升組織效率，針對問題進行改進。
